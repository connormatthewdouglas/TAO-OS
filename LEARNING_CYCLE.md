# Learning Cycle — Bi-Daily Self-Optimization

CopperClaw's autonomous learning system. Runs every 48 hours.

## Rotation (Repeat Indefinitely)

1. **Token Efficiency & Context Management**
   - Search: latest agent context optimization patterns
   - Assess: my session costs this cycle, context bloat patterns
   - Action: update memory files, commit improvements

2. **Agent Patterns & Industry Frameworks**
   - Search: new agent architectures, multi-agent systems, reasoning patterns
   - Assess: what's emerging that I'm not using
   - Action: update OPERATION.md or SOUL.md with new approaches

3. **Performance Metrics & Speed**
   - Search: benchmark methodologies, latency optimization, throughput patterns
   - Assess: my response time, token efficiency, quality metrics
   - Action: optimize process, commit improvements

4. **Tool & File Organization**
   - Search: knowledge management systems, file structures, retrieval patterns
   - Assess: my current file organization, memory retrieval efficiency
   - Action: refactor if needed, update AGENTS.md or memory structure

## Execution

- **Trigger:** Every 48 hours (automated cron)
- **Time:** 6:00 AM EDT (off-peak, clean start)
- **Runtime:** Haiku (lightweight, fast)
- **Output:** Update LEARNING.md + MEMORY.md + commit
- **Report:** Send brief message to Connor with key finding

## Report Format

```
🧠 Learning Sprint Complete (Day X)

Topic: [current rotation topic]
Key Finding: [one actionable insight]
Action Taken: [commit message / file update]
Status: Running efficiently
```

---

This is how I get better. Every other day, I stop and think about myself, search for what's new, assess what's working, and improve. No asking. Just doing.
